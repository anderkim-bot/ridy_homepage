# RIDY 서비스 기술 스택 및 아키텍처 제안서 (V1.0)

## 1. 트래픽 분석 (Framer Analytics 기반)
제공해주신 최근 30일간의 데이터를 바탕으로 분석한 결과입니다.

*   **방문 규모**: 순 방문자 5K / 페이지뷰 29K (월간 약 3만 뷰)
*   **주요 유입 경로**: 네이버(모바일/데스크톱) 검색 유입이 약 70% 이상으로 압도적입니다. -> **SEO(검색 엔진 최적화)가 사업의 핵심**입니다.
*   **기기 비중**: 모바일 사용자가 데스크톱 대비 4배 많습니다. -> **Mobile-First 성능**이 매우 중요합니다.
*   **관심 페이지**: `/rental`, `/bike-model-*`, `/lease-transfer` 순으로 실물 기종과 리스 정보에 집중되어 있습니다.

---

## 2. 추천 개발 스택 (The Scalable Stack)
현재의 프레이머 기반 운영 방식에서 나타나는 이미지 저장 용량 제한과 트래픽 대응력을 완벽히 해결하기 위한 '라이디 맞춤형' 스택입니다.

### **A. Frontend: Next.js (App Router)**
*   **이유**: 단순 React 대비 SEO 성능이 탁월합니다. 네이버/구글 검색 엔진이 페이지 내용을 서버에서 미리 읽을 수 있게(SSR/SSG) 해주어 상단 노출에 유리합니다.
*   **장점**: 프레이머와 유사한 빠른 반응속도를 유지하면서도, 복잡한 비즈니스 로직(리스 계산기, 매물 관리 등)을 구현하기에 가장 적합합니다.

### **B. Backend & Database: Supabase**
*   **이유**: 별도의 복잡한 서버 구축 없이도 기업급 데이터베이스(PostgreSQL)를 즉시 사용할 수 있습니다.
*   **핵심 기능**:
    *   **Realtime Database**: 매물 상태 변화를 즉시 반영.
    *   **Storage**: 현재 문제가 되는 **고화질 이미지 저장**을 무제한에 가깝게 수행 (URL 주소 방식).
    *   **Auth**: 추후 라이더 로그인을 위한 인증 시스템 제공.

### **C. Deployment: Vercel**
*   **이유**: Next.js 제작사가 만든 호스팅 플랫폼으로, 트래픽이 급증해도 서버가 자동으로 확장(Auto-scaling)됩니다.
*   **장점**: 월 3만 뷰는 물론, 갑작스러운 광고 집행으로 방문자가 100배 늘어나도 사이트가 절대 죽지 않습니다.

## 3. 리스 승계 시스템 전환 계획 (Succession Page)

| 구분 | 현재 (Prototype) | 전환 후 (Production) |
| :--- | :--- | :--- |
| **저장소** | 브라우저 LocalStorage (5MB 제한) | **Supabase Storage (수 TB 가능)** |
| **데이터 유지** | 브라우저 청소 시 삭제됨 | **데이터베이스 상에 영구 보존** |
| **이미지 처리** | 원본 그대로 저장 (용량 초과 발생) | **이미지 최적화 처리 (Next.js Image Optimization)** |
| **트래픽 대응** | 개인 PC 사양에 의존 | **글로벌 CDN을 통한 전 세계 초고속 로딩** |

---

## 4. 단계별 도입 로드맵
1.  **1단계 (Infrastructure)**: Supabase 프로젝트 생성 및 이미지 스토리지 설정.
2.  **2단계 (Data Migration)**: 현재 LocalStorage에 저장하던 매물 등록 로직을 Supabase DB 연동으로 교체.
3.  **3단계 (Frontend)**: Next.js 환경으로 이전하여 네이버 검색 노출 극대화 작업 (Metadata 작업).
4.  **4단계 (Optimization)**: 모바일 환경에서 이미지 로딩 속도를 줄이기 위한 WebP 변환 적용.

---

**의견**: 현재 라이디의 가장 큰 위협은 '이미지 저장 공간 부족'으로 인한 데이터 유실과 네이버 검색 유입 의존도에 대한 SEO 대응입니다. 위 스택은 월 100만 명의 사용자가 들어와도 견딜 수 있는 업계 표준 사양이며, 라이디의 비즈니스 성장에 따라 유연하게 확장 가능합니다.
