# RIDY 웹사이트 리뉴얼 프로젝트 브리핑

## 1. 현재 사이트 분석 ([www.ridy.co.kr](http://www.ridy.co.kr/))

### 사이트 구조

`메인 페이지 (/)
├── 회사 소개 (/ridy)
├── PAYOUT (/payout) - 정산 앱 서비스
├── RENTAL (/rental) - 렌탈 서비스
│   ├── 렌탈 기종 보기 (/bike-model-honda)
│   └── 리스 승계 보기 (/lease-transfer)
├── SERVICE CENTER (/service_center) - 서비스 센터
└── 공지사항 (/notification)`

### 주요 페이지별 정보

**메인 페이지**

- 3가지 핵심 서비스 소개 (렌탈/정산/정비)
- "배달라이더와 운영자를 위한 올인원 플랫폼" 포지셔닝
- CTA: 리스/렌탈 문의, 정산앱 문의, 서비스센터 가맹점 문의
- 플로팅 버튼: 문의하기, 차종보기, 카톡상담

**PAYOUT (정산앱)**

- 복잡한 정산 관리 솔루션

**RENTAL (렌탈)**

- 배달 오토바이 렌탈 서비스
- 렌탈 기종 보기 기능
- 리스 승계 서비스

**SERVICE CENTER**

- 라이더 전용 장비 정비 솔루션
- 서비스센터 가맹점 창업 문의

### 현재 사이트의 문제점

- 모바일 최적화 부족
- 정보 구조가 명확하지 않음
- 신뢰감을 주는 비주얼 요소 부족
- CMS 시스템 필요 (공지사항, 렌탈 기종 관리 등)

## 2. 벤치마크 사이트 분석 (arentalnservice.com)

### 장점 및 적용 포인트

**구조적 강점**

- 명확한 메뉴 구조: 회사소개 / 서비스안내 / 렌탈기종 / 렌탈문의 / 공지사항 / 보험사 확인
- 전국 네트워크 강조 (548개 서비스센터)
- 신뢰도 지표 명시 (시장점유율 1위, since 2004)
- 렌탈 제품을 비주얼 중심으로 전시

**UI/UX 요소**

- 대형 히어로 섹션 with 핵심 메시지
- 숫자로 증명하는 강점 (네트워크 개수, 업력 등)
- 제품 슬라이더/캐러셀 활용
- 퀵 메뉴 (렌탈문의, 렌탈기종, 보험확인, 전화)
- 팝업 배너 활용

**기능적 요소**

- 온라인 문의 시스템
- 중고차 매물 확인
- 보험사 확인 기능
- 서비스센터 위치 검색
- 렌탈 절차 안내

## 5. 주요 페이지별 요구사항

### 메인 페이지

**히어로 섹션**

- 강력한 헤드라인: "당신의 첫 라이딩을 책임지는 곳"
- 배경: 실제 라이더가 배달하는 모습 (동영상 또는 고품질 이미지)
- 주요 CTA: 렌탈 문의하기, 차종 보기, 카톡 상담

**신뢰 지표 섹션**

- 전국 서비스센터 개수
- 누적 계약 건수
- 파트너 배달 플랫폼 로고 (배민, 쿠팡이츠 등)
- 고객 만족도

**3가지 핵심 서비스**

- RIDY Rental: 배달 오토바이 렌탈
- RIDY Payout: 정산 관리 앱
- RIDY Service Center: 전문 정비 서비스
- 각 서비스별 아이콘, 짧은 설명, "자세히 보기" 링크

**인기 렌탈 기종 슬라이더**

- 주요 기종 6-8개
- 이미지, 모델명, 월 렌탈료, "상세보기" 버튼

**고객 후기**

- 실제 라이더 후기 (사진, 이름, 평점, 코멘트)
- 3-4개 로테이션

**빠른 문의**

- 이름, 연락처, 문의 내용 간단 입력 폼

### 렌탈 기종 페이지

- 카테고리별 필터링 (브랜드, 배기량, 가격대)
- 그리드 레이아웃
- 각 기종별 카드: 이미지, 모델명, 주요 스펙, 월 렌탈료
- 상세 페이지: 큰 이미지, 전체 스펙, 렌탈 조건, 문의하기 버튼

### 서비스 안내 페이지

- 렌탈 절차 스텝바이스텝
- 각 서비스 상세 설명 (Rental/Payout/Service Center)
- 서비스센터 지도 (전국 네트워크)
- 검색 기능 (지역별 서비스센터 찾기)

### 렌탈 문의 페이지

- 문의 폼: 이름, 연락처, 이메일, 희망 기종, 지역, 문의 내용
- 카카오톡 상담 버튼
- 전화 상담 안내
- 운영 시간 명시

## 6. 디자인 가이드라인

### 컬러 스킴

- 1. Primary & Point Color (핵심 액션 및 강조)
    - **Ridy Vivid Blue (`#4A61ED`)**: 이미지 속 '익일출금' 버튼과 '정산 예정 금액' 타이틀에 사용된 메인 컬러입니다. 신뢰감과 역동성을 동시에 주며, 가장 중요한 **CTA(Call to Action) 버튼**에 사용합니다.
    - **Point Indigo (`#3A4CC0`)**: 블루 계열 내에서 깊이감을 줄 때나 버튼의 Hover(마우스 오버) 상태에 사용합니다.
- 2. Semantic Colors (상태 및 정보 구분)
    - **Text Black (`#1A1A1A`)**: 금액(432,617원)과 같이 가장 중요한 수치 정보를 전달할 때 사용합니다. 완전한 블랙보다는 약간의 깊이가 있는 차콜 블랙이 눈의 피로를 덜어줍니다.
    - **Sub Text Grey (`#767676`)**: '정산 예정일', '누적 포인트' 등 보조적인 설명 텍스트에 사용합니다.
    - **Accent Red/Orange (Optional)**: 이미지에는 없지만, '미납'이나 '긴급 정비' 등 주의가 필요한 요소에 사용할 보조색으로 대비되는 난색 계열을 한 가지 추가하는 것을 추천합니다.
- 3. Background & Surface (배경 및 카드)
    - **Soft Grey Background (`#F7F8FA`)**: 전체 웹사이트의 배경색입니다. 순백색(`#FFFFFF`)보다 카드 컴포넌트를 시각적으로 더 돋보이게 만듭니다.
    - **Card White (`#FFFFFF`)**: 개별 정보가 담긴 카드 영역입니다. 은은한 그림자(Drop Shadow)를 더해 입체감을 줍니다.
    - **Chip Grey (`#F1F2F4`)**: '주정산 차감 예정 금액' 영역처럼 정보를 그룹화하는 칩(Chip) 형태의 배경에 사용합니다.

---

- **Dark Mode 고려**: 라이더들이 야간에 많이 접속하므로, 위 컬러들을 반전시킨 다크모드 대응값을 미리 설정할 것을 요청하세요.
- **Accessibility (웹 접근성)**: 배경 대비 텍스트 명도 대비를 최소 **4.5:1** 이상으로 유지하여 낮 시간 야외에서도 금액 확인이 명확해야 합니다.
- **Point Icon**: 이미지 우측 하단의 포인트 아이콘처럼, 브랜드 컬러의 농도(Opacity)를 조절한 그라데이션을 활용해 세련미를 더해달라고 요청하세요.

### 타이포그래피

- 한글/영문: Pretendard
- 명확한 폰트 사이즈 위계 (H1-H6, Body, Caption)

- 고품질 오토바이 제품 사진

- 밝고 긍정적인 분위기

### 컴포넌트

- 라운드 버튼 (친근함)
- 카드 UI (정보 그룹핑)
- 아이콘 활용 (직관적 이해)
